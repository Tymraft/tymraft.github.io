    • --- {% include brd/cover.html title="Payment Gateway Integration" subtitle="Replacing the legacy NAB Transact integration with Stripe" id="BRD-2026-001" status="draft" version="0.1" author="Tymraft Team" created="2026-06-28" %} {% capture overview_content %}

    This document outlines the business requirements for migrating from the existing NAB Transact payment gateway to Stripe. The migration is driven by the need to support modern payment methods, reduce transaction failure rates, and simplify PCI compliance.

    Stripe's native support for Apple Pay, Google Pay, and card vaulting will unlock checkout improvements that are not possible on the current provider.

    {% include brd/callout.html type="info" content="This BRD covers the web checkout flow only. Native mobile app payment changes are tracked separately under BRD-2026-002." %} {% endcapture %} {% include brd/section.html id="overview" num="1" title="Overview" content=overview_content %} {% capture objectives_content %}
    • Reduce payment failure rate from 4.2% to under 1.5%
    • Support Apple Pay, Google Pay, and BPAY
    • Eliminate annual PCI DSS audit overhead via Stripe's SAQ-A compliance path
    • Deliver a faster checkout experience with one-click payments for returning customers
    {% endcapture %} {% include brd/section.html id="objectives" num="2" title="Objectives" content=objectives_content %} {% capture scope_content %}

    In scope

    • Web checkout integration with Stripe Payment Element
    • Card vaulting for returning customers
    • Apple Pay and Google Pay on supported browsers
    • Webhook handling for payment events
    • Refund and dispute management via Stripe Dashboard

    Out of scope

    • Native iOS and Android in-app payments
    • Subscription or recurring billing
    • Multi-currency support (future phase)
    {% endcapture %} {% include brd/section.html id="scope" num="3" title="Scope" content=scope_content %} {% capture requirements_content %}

    Functional requirements

    IDRequirementPriority
    FR-01 Customers must be able to pay by credit/debit card at checkout High
    FR-02 Returning customers must be offered one-click payment with a saved card High
    FR-03 Apple Pay and Google Pay must be available on supported devices Medium
    FR-04 Failed payments must trigger a retry prompt with a clear error message Medium

    Non-functional requirements

    • Performance: Stripe Payment Element must load within 1.5s on a 4G connection
    • Compatibility: Must support Chrome, Safari, Firefox, and Edge on desktop and mobile
    • Security: Card data must never touch Tymraft servers — handled entirely by Stripe.js
    • Accessibility: Payment form must meet WCAG 2.1 AA
